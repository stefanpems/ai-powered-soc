#Name: 
GetTopIncidentsOwners

#Description: 
Retrieves the list of top owners for incidents created within the specified period.
The summary can optionally be filtered by severity, status, and classification. Allowed values (case insensitive):
- severities: "High", "Medium", "Low", "Informational" 
- status: "New", "Active", "Closed" 
- classification: "TruePositive", "FalsePositive", "BenignPositive", "Undetermined", "" (where "" means unclassified)

#Parameters:
- start_date
  A date-time value in YYYY-MM-DDTHH:MM:SSZ format defining the beginning of the time range for the query. 

- end_date
  A date-time value in YYYY-MM-DDTHH:MM:SSZ format defining the end of the time range for the query. 

- csv_of_severities
  Comma separated list of the severity levels to include. Allowed values (case insensitive): "High", "Medium", "Low","Informational". Include the opening and closing double quotes for each value, and separate each value with a comma. 

- csv_of_status
  Comma separated list of the status to include. Allowed values (case insensitive): "New", "Active", "Closed". Include the opening and closing double quotes for each value, and separate each value with a comma. 

- csv_of_classifications
  Comma separated list of the classifications to include. Allowed values (case insensitive): "TruePositive", "FalsePositive", "BenignPositive", "Undetermined", "". Include the opening and closing double quotes for each value, and separate each value with a comma. Use "" to include unclassified items.

- top_results_number
  An integer specifying how many top results to return.
